# Проект lan-install-interactive

Интерактивный веб-интерфейс для компании «lan-install.ru», специализирующейся на проектировании и монтаже слаботочных систем (СКС, видеонаблюдение, СКУД, ВОЛС). Проект представляет собой современное одностраничное приложение (SPA) с инструментами расчета стоимости и демонстрацией портфолио.

## Основные технологии и архитектура

- **Frontend**: React 19, TypeScript
- **Сборка**: Vite 6
- **Стилизация**: Tailwind CSS
- **Иконки**: Lucide-React
- **Графики**: Recharts (используются в калькуляторе)
- **Роутинг**: React Router v7 (HashRouter)
- **Управление состоянием**: React Hooks (useState, useMemo)

## Структура проекта

- `App.tsx`: Основной компонент с конфигурацией маршрутов.
- `pages/`: Содержит основные страницы приложения:
    - `Home.tsx`: Главная страница.
    - `Calculator.tsx`: Интерактивный калькулятор услуг с возможностью экспорта сметы в CSV.
    - `Portfolio.tsx`: Список выполненных проектов с фильтрацией по категориям.
    - `ProjectDetail.tsx`: Детальная информация о конкретном проекте.
    - `Contact.tsx`: Контактная информация.
- `components/`: Общие UI-компоненты (Header, Footer).
- `src/data/projects.ts`: Статическое хранилище данных о проектах в портфолио.
- `services/geminiService.ts`: Заглушка для интеграции с Gemini AI (в текущей версии функции ИИ отключены).
- `types.ts`: Общие определения типов TypeScript.

## Особенности разработки и конвенции

- **Стилизация**: Используется Tailwind CSS для быстрой и консистентной верстки. Основные цвета бренда (lanBlue, lanOrange, lanGreen) определены в конфигурации (вероятно в `tailwind.config.js` или `index.html`).
- **Роутинг**: Используется `HashRouter`, что позволяет корректно работать при деплое на статические хостинги (например, GitHub Pages) без дополнительной настройки сервера.
- **Данные**: Проекты портфолио хранятся как статический массив объектов в `src/data/projects.ts`. Каждому проекту соответствует категория (`SCS`, `Video`, `Access`, `Fiber`).
- **Калькулятор**: Логика расчета стоимости и состав услуг находятся в `pages/Calculator.tsx`. Цены ориентировочные и могут быть изменены в коде.

## Стандарты отчетности (ASSIGNMENTS.md)

При фиксации выполненных задач в `ASSIGNMENTS.md` необходимо разделять токены на входные (In) и выходные (Out) по следующим причинам:
1. **Прозрачность цены**: Выходные токены — это результат «интеллектуальной» работы (код, тексты). Входные — это «память» (контекст проекта, история логов загрузки).
2. **Контроль контекста**: Если Input растёт слишком быстро, это сигнал, что сессию пора «чистить» или архивировать, чтобы не переплачивать за пересылку одних и тех же файлов в каждом сообщении.
3. **Отчетность перед коммитом**: Перед каждым коммитом необходимо составлять детальный отчет об изменениях и записывать его в файл (например, `COMMIT_REPORT.md`).

## Текущий статус ИИ (Gemini)

Функции искусственного интеллекта в приложении на данный момент **отключены**. Файл `services/geminiService.ts` содержит заглушку. Для включения может потребоваться настройка `GEMINI_API_KEY` в `.env.local` и восстановление логики в компонентах.
