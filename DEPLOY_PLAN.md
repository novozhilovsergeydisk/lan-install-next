# План деплоя проекта lan-install.ru

## 1. Подготовка сервера (✅ Выполнено)
- **Открытые порты:** Проверены. В системе заняты порты 5001, 5002, 5005, 3001 (Node.js), а также 80, 443, 22 и БД. Стандартный порт `3000` свободен, поэтому приложение Next.js будет запущено на нем (или на другом свободном порту, например 3002).
- **Каталог проекта:** В `/var/www/` нет директории для нового домена, поэтому будет создана новая: `/var/www/lan-install.ru`.
- **Резервное копирование Nginx:** Конфигурации Nginx успешно скопированы на сервере.

## 2. Локальная подготовка и отправка
- Анализ и коммит текущих локальных изменений (редизайн калькулятора, правки Header).
- Загрузка исходного кода на сервер в папку `/var/www/lan-install.ru` (через `git` или `rsync`).

## 3. Сборка и запуск на сервере
- Установка зависимостей: `npm install`.
- Сборка production-версии: `npm run build`.
- Запуск сервиса: настройка демона (например, через `systemd`, так как PM2 не используется) для управления процессом `npm run start`.

## 4. Настройка веб-сервера (Nginx)
- Создание конфигурации `/etc/nginx/sites-available/lan-install.ru`.
- Настройка проксирования (`proxy_pass`) запросов с порта 80/443 на порт запущенного приложения (например, `http://127.0.0.1:3000`).
- Включение сайта через символическую ссылку в `sites-enabled` и перезапуск Nginx.

## 5. Выпуск SSL-сертификата (HTTPS)
- Использование Certbot для генерации сертификата: `sudo certbot --nginx -d lan-install.ru -d www.lan-install.ru`.
